# Versioning

## Contract versioning

The NeuroCAD public API is intended to evolve through explicit versioning at the contract layer.

The initial draft surface is published under:

- `v1`

## Compatibility stance

The public goal is:

- additive changes for non-breaking expansion
- explicit version bumps for breaking changes
- release notes for externally visible contract updates

## Artifact versioning

Where applicable, public artifacts should expose:

- contract version
- release date
- schema version
- deprecation state

## Current status

Initial public draft.

This repository is the canonical location for externally visible API contract changes.
